# Portfolio Game — Project Summary

**Project:** 2D game portfolio site at `/Users/luoph/Desktop/portfoliov2`. Static, no backend, free hosting (leaning Vercel; GitHub Pages fallback).

**Stack:** Phaser 3 + Vite 8 (chosen over Godot to avoid its ~25MB WASM load). Node 25. `npm run dev` → localhost:5173. Build is green (~326 KB gzipped; art served from `public/`, not bundled). 0 npm vulnerabilities.

## Display / scaling

Game fills the whole window at a **constant pixel scale** — a bigger window shows MORE world, a smaller one less (no letterboxing). Implemented via `Phaser.Scale.NONE` + `zoom: PIXEL_SCALE` over a low-res buffer sized to `ceil(window / PIXEL_SCALE)`; `main.js` drives `game.scale.resize(...)` on window resize (rAF-debounced), which auto-resizes each scene's main camera. `config.js`: `PIXEL_SCALE` (currently 3 — bump to 4 for chunkier/less-world), plus `MIN_VIEW_WIDTH/HEIGHT` floors. Screen-pinned UI (dialogue box) re-lays-out via `BaseWorldScene.layoutDialogue()`, bound to the scale `resize` event.

**Concept:** Top-down explorer. Three scenes planned — cabin exterior, cabin interior, dungeon. NPCs/rooms map to specific portfolio domains (blacksmith = systems, alchemist = ML, merchant = web, dungeon boss = capstone); dialogue carries the content.

## Art

The old **Mana Seed** character sheets are NO LONGER USED (their license forbids gen-AI workflows). Current art:

**Player — LPC ("Universal LPC Spritesheet"), `public/assets/player/`.** One sheet per animation (`walk.png`, `idle.png`, … + unused `run/jump/slash/…`). Each is 832×256 = **64px frames, 13 cols × 4 rows**. Direction → row: `up=0, left=1, down=2, right=3`. Walk = cols 1–8 (col 0 = standing); idle sheet = cols 0–1 (breathing). Single composited character — NOT a paper-doll. `charAnims.js` registers `player-walk-<dir>`/`player-idle-<dir>`, pulling idle frames from idle.png and walk frames from walk.png (a sprite auto-swaps texture per frame).

**Interior tileset — Kenney "Roguelike/RPG Indoor", `public/assets/interior/roguelikeIndoor_transparent.png`.** 458×305, **16px tiles with 1px spacing → 27 cols × 18 rows (rows 0–17; there is NO row 18)**. Loaded as spritesheet key `'indoor'` (`spacing:1`). Frame index = `row*27 + col` (helper `tileFrame(col,row)` in `Interior/constants.js`, `INDOOR_COLS=27`). NB the user references tiles as **(row, col)** and is sometimes off-by-one — verify against the 0–17 row range. `TILE` is now **16** (matches the art's native size; see Display).

NPCs/buildings/dungeon are still placeholder blobs/rects (no themed art yet).

## Done

- **`src/entities/charAnims.js`** — `registerCharAnims(scene, key, {idleKey, walkKey})` builds idle/walk anims from the LPC layout (see Art).
- **`src/entities/Player.js`** — single composited LPC sprite (texture `player_idle`, anims swap to `player_walk`), WASD/arrows, 4-directional. Paper-doll overlays/`syncLayers` removed; depth via `applyDepth()`. `?facing=down|up|left|right` debug URL hook. Scaled by `PLAYER.scale` (0.5) so the 64px LPC frame is ~2 tiles tall at `TILE=16`; the Arcade body scales with the sprite, so the feet collider stays correct. `PLAYER.speed`=70.
- **Entity object hierarchy (`src/entities/`)** — `GameObject` (top-level base: owns size `x,y,w,h`, **display** = `tiles` [spritesheet frames stacked top→bottom, one per tile-row] / `texture`+`frame` [a single png stretched to w×h] / flat `color`, and a whole-footprint `solidRect` collider; depth-sorts by base-y). `InteriorObject extends GameObject` — adds tile-coord defs (`tx/ty/tw/th`), a **base-only** footprint collider (`PROP_FOOT`, so tall props can be walked behind), decor depth offset, and the `[E] look` `interactable()`. **Objects own their dialogue**: `this.examine` (defaults to `def.examine`). Subclasses: **`Bed`** (full-footprint collider so all sides block; owns its examine line), **`Table`** (full-footprint collider; `interactable()` anchored at centre with a range reaching one tile past every edge so the prompt triggers from any side), **`Chair`** (forces `solid:true`), **`Mirror`** (`interactable()` action calls `this.open(scene)` which invokes the **global `openAboutModal`** directly + freezes via `scene.modalOpen` — `BaseWorldScene.openAbout` was removed). `InteriorScene` maps `def.type` → class via an `OBJECT_TYPES` registry (`{table,mirror,chair,bed}`), else `Mirror` for `about:true`, else `InteriorObject`.
- **Interior floor + side borders** — the whole interior floor tiles the red indoor tile (frame 24); left/right room edges overlay bordered tiles (77/79) via `InteriorScene.createSideBorders()`. `createFloor` prefab extended with a `tile:{sheet,frame}` source (bakes one spritesheet frame into its own TILE texture so tiling doesn't bleed neighbours). Per-room `floor/floorAlt` colours in `interiors.js` are now unused (dead fields).
- **`src/scenes/BaseWorldScene.js`** — shared base for all walkable scenes: interaction prompt, dialogue box, nearest-interactable scan, and fade scene transitions. Scenes just build their world, make the player, and push `{x,y,promptY,range,label,action}` into `this.interactables`.
- **`src/scenes/CabinExterior/CabinExteriorScene.js`** (own folder; layout consts in sibling **`constants.js`**) — **Pokémon-style 2D town** (26×18 tiles, 832×576) ringed by a textured, solid **forest border**. **Home** top-left, **Dungeon** (cave mouth in a rocky outcrop) bottom-right, shops between: **Smithy** top-right, **Market** center, **Apothecary** bottom-left. **Every door is on the building's bottom edge** (uniform face — old north/south `face` logic removed; buildings depth-sort by base, player passes in front when below; each has a doormat). A cosmetic **path network** (`PATHS` array, drawn as shoulder + surface) forms a loop every door opens onto, ending at the dungeon. `[E] enter` fades to the matching interior; returning spawns on that doorstep. Layout constants up top: `BUILDINGS` (tile `col/row/w/h`), `PATHS`, `COLS/ROWS`, `CAVE`.
- **`src/scenes/Interior/InteriorScene.js`** (own folder; consts in sibling **`constants.js`**) — one generic, data-driven interior (key `'Interior'`, restarted with `{id}`). Renders floor/walls with a bottom doorway, props (via the entity object hierarchy above), and the resident NPC; `[E] leave` returns to the exterior (or, for `home`, plays its `lockedExit` message — exit currently sealed). Also renders the dungeon. **Deeper top/back wall**: `createWall` prefab takes `topThickness` (`BACK_WALL` = `TILE*2`) so the top wall is deeper than the sides/bottom (`WALL`=14); `shadeBackWall()` makes it read as a real wall — a top→bottom shading gradient, a darker crown band along the very top, a faint sheen line, and a light-grey baseboard at the wall→floor change. `createSideBorders()` overlays grid-snapped wood-edge tiles (`tileFrame(23,3)` left / `tileFrame(25,3)` right) starting flush below the back wall. `centerCamera()` keeps the room centred.
- **`src/data/interiors.js`** — room defs (home + 3 shops + **dungeon**) in tile coords: floor/wall colors, props, and `resident` (links to `NPCS`). Prop fields: `tx/ty/tw/th` (fractional allowed), display (`tiles`/`texture`+`frame`/`color`), `type` (→ object class), `name`+`examine`, `wares`, `about`, `solid`/`decor`. A `wares` prop auto-lists that NPC's projects; dungeon "boss" examines into `DUNGEON_BOSS`. **Home is themed with real tile art** (shops/dungeon still color blobs): a `bed` (texture frame, top-right), the `mirror` (2-frame stack, opens About), two wall-decoration strips left of the mirror (rows 17 & 16, cols 16–18, walk-over decor), a `table` (2-frame stack) flanked by two `chair`s. Old Notice Board / Chest / rug removed.
- NPCs live **inside** their shops (still placeholder blobs); `[E] talk`. Dungeon boss is a placeholder block for now.
- **`src/ui/textStyles.js`** — shared canvas TextStyle presets (`LABEL`, `SIGN`, `SIGN_DANGER`) used by both scenes.
- **`src/ui/aboutModal.js`** + **`src/style.css`** — the **Home Mirror** `[E]` opens an HTML/DOM "About Me" overlay (photo + bio + links, Esc/✕/backdrop to close); player frozen via `scene.modalOpen`. The `Mirror` entity owns the interaction — it calls the global `openAboutModal` directly with `PROFILE` and sets/clears `modalOpen` (no longer routed through a `BaseWorldScene` method). `style.css` (extracted from `index.html`) is imported in `main.js`.
- **Game starts in the Home interior** (`PreloadScene` → `Interior {id:'home'}`); the home exit is sealed (`lockedExit`).
